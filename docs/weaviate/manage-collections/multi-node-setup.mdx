## Cluster Resharding

Weaviate allows you to dynamically redistribute data across nodes using the resharding feature. This enables you to optimize your cluster's performance and scalability without downtime.

### Resharding Characteristics

- Resharding is an asynchronous operation that redistributes data across nodes
- Provides automatic load balancing during migration to prevent node overload
- Configurable batch sizes and migration speed throttling
- Real-time consistency checks to ensure data integrity
- Rollback support in case of migration failures

### Operational Considerations

- During resharding, read operations continue normally
- Write operations are temporarily queued and replayed after migration completes
- Typical use cases include:
  - Scaling up clusters under heavy load
  - Rebalancing after node additions/removals
  - Optimizing shard distribution for query performance patterns

:::tip Monitoring
Use the dedicated monitoring endpoints to track resharding progress and ensure a smooth transition.
:::

### API Endpoint

The resharding operation can be triggered via the `/v1/cluster/resharding` REST API endpoint. Refer to the [API documentation](/weaviate/api/rest) for detailed parameter specifications.
